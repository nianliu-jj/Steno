Rust 生命周期笔记

基本概念

生命周期（Lifetime）是 Rust 中引用保持有效的作用域。编译器通过生命周期来确保所有借用都是有效的。

大多数情况下生命周期是隐式的、可推断的，就像类型一样。当有多种可能的类型时，我们必须注明类型；同样，当引用的生命周期可能以不同方式关联时，我们必须注明生命周期。

生命周期标注语法

&i32        // 引用
&'a i32     // 带有显式生命周期的引用
&'a mut i32 // 带有显式生命周期的可变引用

函数中的生命周期

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

关键点：

  • 返回值的生命周期等于参数中较短的那个
  • 编译器无法自动推断时需要手动标注
  • 生命周期标注不改变引用的实际存活时间

结构体中的生命周期

struct ImportantExcerpt<'a> {
    part: &'a str,
}
任何持有引用的结构体都必须标注生命周期，这确保结构体不会比其引用的数据存活更久。
