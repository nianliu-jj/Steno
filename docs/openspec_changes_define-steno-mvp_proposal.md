# 变更提案：定义 Steno MVP

## 为什么

Steno 当前已有高保真原型和品牌方向，但需要形成可交付的需求规格，作为后续 Rust/Tauri 开发、任务拆分和验收的依据。

用户希望 Steno 成为一款简约但功能强大的速记笔记软件，参考 FloatMemo 的状态栏和浮窗体验，同时强化剪贴板历史、无限画布、本地优先和 Zen 写作。

## 做什么

- 定义 Steno MVP 的核心能力边界。
- 明确状态栏、浮窗速记、全局快捷键、剪贴板历史、置顶笔记、无限画布、主题、本地存储、Markdown 导出和 Zen 模式的需求。
- 明确首发平台为 macOS 桌面端。
- 明确默认本地存储，不做云同步和账号系统。
- 为后续实现提供验收标准。

## 不做什么

- 不做多人协作。
- 不做云同步。
- 不做移动端。
- 不做 AI 自动总结或联网生成。
- 不做插件市场。
- 不在 MVP 中支持复杂媒体笔记。

## 影响

- 产品：从普通笔记编辑器收敛为“桌面速记层”。
- 设计：继续沿用温暖克制、macOS 原生感、便签纸感的方向。
- 工程：Rust/Tauri 应用需要处理 macOS 状态栏、全局快捷键、剪贴板监听、本地数据库和导出。
- 隐私：默认本地保存，后续任何同步能力都必须单独提案。
