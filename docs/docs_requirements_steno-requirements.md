# Steno 需求文档

## 1. 背景

Steno 是一款使用 Rust 技术栈开发的 macOS/Windows/Linux 速记笔记软件。它参考 FloatMemo 的“随处书写”体验，但产品重点更偏桌面效率：常驻状态栏、全局快捷键呼出、浮窗速记、置顶便签、无限画布、本地存储和 Zen 写作。

Steno 的核心判断是：很多笔记不是从“打开一个笔记应用”开始，而是在用户看视频、写代码、开会、聊天、浏览网页时突然出现。软件应当成为当前工作流上方的一层轻量捕获层，帮助用户先记录，再整理，最后输出为 Markdown。

## 2. 产品目标

- 让用户在不切换应用的情况下完成快速记录。
- 自动保存剪贴板历史，降低“刚才复制过什么”的回溯成本。
- 用置顶笔记和无限画布承接短期记忆、待办、常用片段和学习资料。
- 所有数据默认只保存在本地，建立隐私可信感。
- 通过 Zen 模式把零散素材整理成可读、可导出的 Markdown 文档。

## 3. 目标用户

- 经常看学习视频、阅读文档、写代码并需要随手记录的人。
- 依赖剪贴板复制代码、链接、回复模板、资料片段的重度桌面用户。
- 不想把私人笔记同步到第三方服务器的本地优先用户。
- 喜欢便签式组织，但又需要搜索、标签、Pin 和导出的效率用户。

## 4. 产品定位

Steno 不是大型知识库，也不是协作文档工具。它是一个本地优先的“桌面速记层”：

- 捕获层：状态栏、浮窗、全局快捷键、剪贴板监听。
- 整理层：搜索、标签、Pin、无限画布、智能排列。
- 写作层：Zen 模式、Markdown 导出。

## 5. 范围

### 5.1 MVP 必须包含

- macOS 状态栏常驻入口。
- 全局快捷键呼出浮窗。
- 浮窗速记并自动保存。
- 剪贴板历史记录、搜索、Pin 和标签。
- 置顶笔记面板。
- 无限画布上的便签卡片拖拽、缩放、平移和智能排列。
- Light / Dark / System 主题切换，以及基础强调色选择。
- 本地存储和 Markdown 导出。
- Zen 写作模式。

### 5.2 MVP 暂不包含

- 多端云同步。
- 多人协作。
- AI 自动总结、语义搜索或联网生成。
- 图片、音频、视频等复杂媒体笔记。
- 插件市场。
- Windows 和 Linux 发行版。

## 6. 关键用户故事

### US-01 状态栏待命

作为桌面用户，我希望 Steno 安静常驻在 macOS 状态栏中，这样我不需要在 Dock 或应用切换器里寻找它。

验收标准：

- 应用启动后状态栏出现 Steno 图标。
- 点击图标可打开速记浮窗或主菜单。
- 用户可以设置开机自启动。
- 状态栏入口在无操作时不弹通知、不抢焦点。

### US-02 全局快捷键呼出浮窗

作为正在看视频或写代码的用户，我希望按下自定义快捷键即可呼出速记浮窗，这样我不用离开当前应用。

验收标准：

- 默认快捷键为 `Option + S`，用户可修改。
- 快捷键在非 Steno 应用中也能触发。
- 快捷键冲突时提示用户重新设置。
- 浮窗打开后文本输入框自动获得焦点。
- 保存或按 `Esc` 后浮窗自动收起，并将焦点还给原应用。

### US-03 浮窗速记

作为用户，我希望在当前屏幕上直接记录内容，这样不会打断当前任务。

验收标准：

- 浮窗支持纯文本和 Markdown 输入。
- 保存后内容进入 Inbox。
- 用户可在保存前添加标签或 Pin。
- 浮窗可设置为临时置顶。
- 未保存内容在关闭前自动缓存，重新打开时可恢复。

### US-04 剪贴板历史

作为经常复制资料的人，我希望 Steno 自动记录剪贴板文本，这样复制过的链接、代码和文字不会丢失。

验收标准：

- 自动记录文本、链接和代码片段。
- 支持按关键词搜索历史内容。
- 支持按类型、来源应用、时间和标签过滤。
- 支持 Pin 收藏重要剪贴板条目。
- 支持把剪贴板条目发送到画布或转为普通笔记。
- 用户可设置保留时长和最大条数。
- 密码、验证码等敏感内容应提供排除策略或清理入口。

### US-05 置顶笔记

作为用户，我希望重要内容始终显示在显眼位置，这样常用信息和待办不需要反复查找。

验收标准：

- 任意笔记或剪贴板条目都可 Pin。
- 置顶内容在主界面和浮窗中优先展示。
- 置顶列表支持手动排序。
- 取消 Pin 后条目回到原来的时间线或画布位置。

### US-06 无限画布管理

作为喜欢视觉整理的人，我希望在无限画布上自由排列便签，这样可以像真实便签纸一样组织想法。

验收标准：

- 画布支持拖拽移动卡片。
- 画布支持缩放和平移。
- 卡片支持标题、正文、标签、Pin 状态和创建时间。
- 支持智能排列，按标签、Pin 状态或时间自动整理。
- 画布状态本地持久化，重启后保持位置和缩放。

### US-07 主题个性化

作为用户，我希望调整主题和强调色，这样 Steno 能适应不同工作环境和个人偏好。

验收标准：

- 支持 Light、Dark、System 三种模式。
- 支持选择图标或强调色。
- 主题修改即时生效。
- 主题设置本地保存。
- 主题不能影响内容可读性和基本对比度。

### US-08 本地存储与导出

作为重视隐私的用户，我希望所有数据默认只保存在本机，并可导出为 Markdown。

验收标准：

- 默认不上传任何笔记或剪贴板内容。
- 用户可查看本地数据目录。
- 支持按单条笔记、画布分组或全部数据导出 Markdown。
- 导出内容包含标题、正文、标签、创建时间和更新时间。
- 导出失败时给出可理解的错误提示。

### US-09 Zen 模式

作为用户，我希望在整理素材时隐藏干扰，只保留写作区域。

验收标准：

- Zen 模式隐藏侧边栏、画布、剪贴板和主题面板。
- 页面只保留标题、正文、字数/保存状态和退出入口。
- 支持从画布或 Inbox 选择素材进入 Zen 模式。
- 写作内容自动保存到本地草稿。
- 可从 Zen 模式直接导出 Markdown。

## 7. 信息架构

### 7.1 主导航

- 画布：默认工作区，承载便签卡片和功能主链路。
- 剪贴板：历史记录、搜索、过滤、Pin。
- 置顶：所有 Pin 内容的集中视图。
- 标签：标签管理和按标签浏览。
- 设置：快捷键、主题、本地存储、导出和隐私。

### 7.2 核心对象

- Note：普通笔记或便签。
- ClipboardItem：剪贴板历史项。
- Tag：标签。
- CanvasBoard：画布。
- ThemeSetting：主题设置。
- ExportJob：导出任务。

## 9. 主要流程

### 9.1 快速记录流程

1. 用户按 `Option + S`。
2. Steno 在当前屏幕显示浮窗。
3. 输入框自动聚焦。
4. 用户输入文本，可选标签或 Pin。
5. 用户保存。
6. 内容进入 Inbox，浮窗自动收起。
7. 焦点返回原应用。

### 9.2 剪贴板回溯流程

1. 用户复制文本、链接或代码。
2. Steno 后台记录剪贴板内容。
3. 用户打开主界面或搜索面板。
4. 输入关键词或选择过滤条件。
5. 用户复制回目标内容，或将条目转为笔记。

### 9.3 画布整理流程

1. 用户从 Inbox、剪贴板或浮窗保存内容。
2. 条目生成便签卡片。
3. 用户拖拽卡片组织空间关系。
4. 用户缩放和平移画布。
5. 用户可按标签或 Pin 状态智能排列。

### 9.4 Zen 写作流程

1. 用户选择一组笔记或画布内容。
2. 进入 Zen 模式。
3. Steno 隐藏其他面板，只保留写作区域。
4. 用户整理正文。
5. 自动保存草稿。
6. 用户导出 Markdown。

## 10. 非功能需求

### 性能

- 状态栏驻留时 CPU 占用应接近空闲。
- 浮窗呼出感知延迟目标不超过 150ms。
- 搜索 10,000 条剪贴板历史时应在 200ms 内返回首屏结果。
- 主界面滚动、拖拽和缩放应保持流畅。

### 隐私与安全

- 默认本地存储，不上传服务器。
- 提供清空剪贴板历史入口。
- 支持设置剪贴板保留期限。
- 对可能敏感的来源应用提供排除名单。
- 本地数据库路径应明确可见。

### 可用性

- 常用操作应支持键盘完成。
- 浮窗、搜索、保存、退出 Zen 模式必须有快捷键。
- 主题切换不应影响可读性。
- 错误提示应说明原因和下一步操作。

### 可靠性

- 保存操作应具备失败提示。
- 意外退出后应恢复未保存浮窗草稿。
- 数据迁移需要版本标记和回滚策略。

## 11. 技术约束

- 核心应用使用 Rust 开发。
- 桌面壳建议使用 Tauri，前端可用 Web 技术实现。
- 本地存储建议使用 SQLite。
- 全文搜索可使用 Tantivy 或 SQLite FTS，具体由实现评估决定。
- macOS 权限需要覆盖剪贴板、辅助功能、开机启动和全局快捷键。

## 12. MVP 分期

### Phase 1：捕获闭环

- 状态栏入口。
- 全局快捷键。
- 浮窗速记。
- 本地保存。
- Inbox 列表。

### Phase 2：剪贴板与检索

- 剪贴板历史。
- 搜索和过滤。
- Pin 与标签。
- 清理策略。

### Phase 3：画布与整理

- 便签卡片。
- 拖拽、缩放、平移。
- 智能排列。
- 置顶面板。

### Phase 4：写作与导出

- Zen 模式。
- Markdown 导出。
- 主题设置。
- 本地数据目录与隐私设置。

## 13. 验收总则

- 每个核心功能必须能通过界面完成，不只存在于设置或说明中。
- 保存、搜索、Pin、标签、导出必须有明确反馈。
- 应用离线可用。
- 用户不登录也能使用全部 MVP 功能。
- 不得默认上传任何用户内容。

## 14. 未决策项

- 剪贴板历史默认保留时长：建议 7 天。
- 是否支持图片剪贴板：建议 MVP 暂不支持。
- 是否需要端到端加密：本地优先阶段先做本地数据库和权限控制，后续再评估。
- 首发平台是否限定 macOS：建议 MVP 只做 macOS。
