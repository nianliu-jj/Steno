# 主工作台布局精简设计

## 背景

当前主工作台界面已经有完整的桌面壳结构：顶部自定义标题栏、左侧主导航、主内容区和底部置顶栏。但在 `main` 模式下，界面入口存在两层重复结构：

- 顶部标题栏已经提供全局搜索与窗口级导航。
- 主内容区上方仍然保留大标题与描述区。
- 底部另有一条全宽的“置顶内容”常驻栏。

这三层叠加后，主内容真正开始的位置被明显下压。对“笔记列表”这种高频浏览界面来说，信息入口偏重，卡片区起点偏晚，视觉重心也被顶部标题区与底部置顶栏同时分散。

用户要求移除截图中框选的两个区域，并保持整体美观性。因此本次设计目标不是重做整个主界面，而是在保留现有结构语言的前提下，让 `main` 视图更直接进入内容。

## 目标

- 删除主内容区顶部的大标题/描述区。
- 删除底部“置顶内容”常驻栏。
- 保持顶部标题栏、左侧导航、主内容区三层主骨架不变。
- 让主内容以更轻的工具条起始，直接承接笔记列表。
- 保留“置顶”能力，但不再通过底部常驻栏表达。
- 去掉左侧重复的“搜索”入口，统一通过顶部搜索进入搜索页。

## 不在本次范围

- 不调整顶部标题栏的整体结构。
- 不重做笔记卡片的交互模型。
- 不新增新的筛选逻辑或数据模型。
- 不恢复“置顶内容”到新的全局面板或抽屉。
- 不改动搜索页本身的布局与行为。

## 推荐方案

采用“A. 轻工具条首行”方案。

核心思路：

- 删除 `MainWorkbenchShell` 中的页面标题头部和底部置顶栏。
- 把原本出现在页头右侧的 `MainView` 工具按钮，转为主内容区第一行的轻量工具条。
- 工具条下方直接接笔记卡片网格或空状态。
- 通过稳定的上边距、较低高度控制条和轻分隔线维持信息层级，而不是继续使用大标题块。

这样处理后，用户打开主界面会先看到：

1. 顶栏搜索。
2. 左侧导航。
3. 主内容区内的轻工具条。
4. 紧接其后的笔记卡片内容。

这条路径比当前结构更短，也更符合笔记列表页的高频浏览特征。

## 结构设计

### 1. 应用壳层

`src/App.vue` 继续负责根据 `ui.mode` 渲染顶层视图，但主工作台导航做两处精简：

- `shellNavItems` 去掉 `search` 项。
- `main` 视图不再通过 `#actions` 插槽向 `MainWorkbenchShell` 注入工具按钮。

搜索功能仍保留，通过顶栏搜索框进入 `search` 视图。这样可以消除“左侧搜索”和“顶部搜索”的重复入口。

### 2. 主工作台壳组件

`src/components/MainWorkbenchShell.vue` 保留：

- 自定义标题栏。
- 全局搜索框。
- 左侧导航与底部设置/语言/折叠控制。
- 主内容容器。

`MainWorkbenchShell` 删除：

- `workbench-page-header` 模板结构。
- `footer.bottombar` 模板结构。
- 仅为底部置顶栏服务的 `pinnedChips` 计算逻辑。
- `summarizePinnedNote()`。
- `onMounted` 中 `loadPinned()` 的调用。

删除后，`workbench-main` 将只保留一个更直接的内容承载层，内容由各页面视图自行负责起始布局。

### 3. 主内容页

`src/views/MainView.vue` 维持现有数据和行为，但调整布局职责：

- `compactActions` 模式不再理解为“页头动作区”，而是“内容区首行工具条”。
- 当 `ui.mode === 'main'` 时，工具条显示在内容区顶部。
- 工具条下方直接进入笔记网格。
- 空状态同样以工具条起始，并在其下居中展示空态内容。

工具条保留现有按钮集合：

- `筛选`
- `速记`
- `新建笔记`

其中：

- `新建笔记` 仍为主按钮。
- `速记` 为次级操作。
- `筛选` 保持为轻量控制按钮，即使当前未接筛选逻辑，也作为布局起点存在。

### 4. 置顶能力表达

底部置顶栏删除后，不再显示：

- “置顶内容”标签。
- 空置顶提示“暂无置顶内容”。
- `0/5` 一类容量计数。

置顶能力保留在两处：

- 笔记卡片高亮背景 `paper-1`。
- 笔记卡片内“置顶 / 取消置顶”操作。

这意味着本次是“移除全局置顶展示层”，不是“移除置顶功能”。

## 视觉与交互原则

### 信息层级

- 顶栏负责全局级动作：返回、搜索、窗口控制。
- 左侧导航负责模块切换。
- 主内容区负责当前页面的局部操作和数据展示。

删除页面标题区后，不能再依赖大号标题建立层级，因此层级改由以下元素承担：

- 内容区顶部留白。
- 工具条的稳定位置。
- 工具条与卡片网格之间的固定间距或轻分隔线。
- 卡片网格本身作为主视觉重心。

### 密度与留白

- 内容区顶部留白要明显小于当前页头高度，但不能压缩到贴边。
- 工具条高度维持现状，避免引入新的视觉噪音。
- 工具条与卡片区之间保留稳定间距，让删除页头后界面仍有呼吸感。

### 一致性

- 不引入新的色板。
- 不改变现有卡片语言、按钮圆角和边框体系。
- 保持当前暖色浅底工作台风格。

## 实现落点

### `src/App.vue`

- 删除 `shellNavItems` 中的 `search`。
- 在 `MainWorkbenchShell` 中移除 `ui.mode === 'main'` 时的 `#actions` 注入。
- 保持 `MainView` 正常渲染。

### `src/components/MainWorkbenchShell.vue`

- 删除 `workbench-page-header` 相关模板与样式。
- 删除 `bottombar` 相关模板与样式。
- 删除 `notes` store 依赖、`pinnedChips`、`summarizePinnedNote()` 和 `loadPinned()`。
- 调整 `workbench-main`、`workbench-content` 的内边距与高度逻辑，保证主内容直接起始仍有秩序。

### `src/views/MainView.vue`

- 让 `compactActions` 工具条成为主内容区首行布局。
- 调整 `main-root`、`main-toolbar`、`notes-grid`、`empty-state` 的上下间距，使其适应新结构。
- 不修改数据加载和卡片交互逻辑。

### 测试

需要同步更新：

- `src/components/MainWorkbenchShell.test.ts`
  - 删除底栏相关断言。
  - 更新导航项数量断言。
  - 补充“不渲染页头与底栏”的断言。
- `src/views/MainView.test.ts`
  - 如果测试依赖旧的动作区注入方式，需要改为验证工具条在主内容内可见。
- 其他引用 `search` 导航项数量或页头描述的断言，按新结构收口。

## 错误处理与回归关注点

- 删除 `loadPinned()` 后，要确认主界面不再因底栏依赖而触发无效加载。
- 删除 `search` 侧栏入口后，要确认顶栏搜索仍能可靠进入搜索页。
- 删除页头后，要确认窄屏下工具条换行不会和卡片区重叠。
- 删除底栏后，要确认主内容高度计算正常，不出现底部留空异常或滚动区域错位。

## 验证方式

- 单元测试：
  - `MainWorkbenchShell` 渲染与导航行为。
  - `MainView` 工具条存在性与按钮行为。
- 手动验证：
  - 主界面打开后不再出现标题区与底部置顶栏。
  - 左侧导航不再出现“搜索”入口。
  - 顶栏搜索仍可进入搜索页。
  - 主界面动作按钮位于内容区第一行。
  - 笔记网格与空状态在桌面和窄屏宽度下都保持稳定。

## 自检结果

- 没有占位项或未决描述。
- 范围限定在主工作台布局精简，没有扩散到搜索页、数据模型或置顶功能重构。
- 结构改动与用户确认的 A 方案一致。
- 保留现有视觉语言，只精简层级，不引入新的大组件。
