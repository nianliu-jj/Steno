# 主列表标签筛选与自定义右键菜单设计

## 背景

主窗口的笔记列表当前只展示最近笔记卡片，并保留“筛选、速记、新建笔记”工具栏。用户需要在该列表中按标签筛选文档，并替换浏览器默认右键菜单为应用内自定义菜单。

## 备选方案

方案一：仅在主列表页实现本地标签筛选与自定义菜单。主列表继续加载最近笔记，筛选基于已加载笔记的 `tags` 字段完成，标签编辑、重命名和删除通过现有 `saveNote` / `removeNote` 能力落库。

方案二：把标签筛选接入后端 `search_notes`。筛选结果可覆盖更多历史笔记，但会改变主列表“最近笔记”的现有边界，并需要额外处理“无标签”查询语义。

方案三：抽出全局笔记操作菜单组件，主列表、搜索页、画布页共用。复用性更好，但当前需求只指定主列表，跨页面抽象会放大改动范围。

推荐采用方案一。它最贴合现有主列表结构，风险低，能完整满足本轮交互需求；HTML 导出是唯一需要新增的后端能力。

## 交互设计

筛选按钮点击后显示下拉菜单。菜单内容来自当前已加载笔记的所有标签，按去重后的标签名排序，并固定提供“无标签”选项。每个选项使用复选框；顶部提供“全选”复选框。多选规则为“或”过滤：选中任意标签后，展示包含任一选中标签的文档；同时选中“无标签”时，也展示没有任何标签的文档。全选表示选中所有标签选项和“无标签”，等价于不缩窄列表。

主列表根容器监听 `contextmenu` 并调用 `preventDefault()`，禁用浏览器默认右键菜单。右键点击文档卡片时，菜单绑定该文档；右键点击空白区域时，菜单没有目标文档。菜单项包括：新建、编辑、标签、导出为、打印、重命名、删除。没有目标文档时，编辑、标签、导出为、重命名、删除置灰。

“新建”进入笔记编辑器。“编辑”进入当前文档编辑器。“标签”打开弹框，列出该文档标签，每行一个单行输入，支持添加、修改、删除并确认保存。“重命名”打开弹框，提供一个单行输入修改标题并确认保存。“删除”沿用现有删除逻辑，若文档已置顶则先关闭贴纸窗口。

导出菜单包含 Markdown、Html、PDF。Markdown 和 PDF 复用现有命令；HTML 新增后端命令，把笔记 `htmlContent` 包装成完整 `.html` 文件并写入 `<data_dir>/exports/`。PDF 当前后端仍会返回不可用错误，前端展示错误提示。

打印调用浏览器 `window.print()`，用于打印当前窗口内容；该项不依赖选中文档。

## 数据流

主列表从 `notes.notes` 派生：

- `allFilterOptions`：当前笔记标签去重后加“无标签”。
- `visibleNotes`：基于选中筛选项过滤后的列表。
- `contextTargetNote`：右键菜单当前关联文档，空白区域为 `null`。

标签修改与重命名通过 `notes.saveDraft()` 保存完整笔记字段，并在成功后更新本地 store。删除通过 `notes.removeNote()`。导出通过 `useDb()` 调用对应导出命令。

## 错误处理

保存、删除和导出失败时使用 Naive UI message 展示错误。空标签行会被忽略，重复标签去重。重命名输入为空时保存为空标题，后端会按内容派生标题；前端显示仍使用现有“无标题”兜底。

## 测试

前端单元测试覆盖：

- 筛选菜单展示去重标签、无标签、全选，多选后列表正确变化。
- 右键空白区域与文档卡片时菜单项可用状态不同，并禁用默认菜单。
- 标签弹框支持添加、修改、删除并保存。
- 重命名弹框保存标题。
- 导出 Markdown / Html / PDF 调用对应命令。

Rust 测试覆盖：

- HTML 导出文件名复用现有命名策略。
- HTML 导出写入完整文档结构，并包含标题与已渲染内容。
