## Why

当前主窗口编辑页仍然基于“原始 Markdown 文本输入 + 可选预览”的轻量模型，无法满足“直接在排版结果上编辑”的写作体验，也缺少只读切换、代码模式、大纲悬浮展开、Zen 大纲侧栏与宽度持久化等成体系能力。随着主窗口编辑流已经成为日常整理入口，这些体验缺口已经直接影响主编辑页与 Zen 模式的一致性和可用性，因此需要把编辑模型与页面壳层一起提升到下一阶段。

## What Changes

- 将主编辑页与 Zen 页统一为“三态编辑模型”：排版编辑态、只读态、代码态。
- 将主编辑体验改为默认进入排版编辑态，用户直接在渲染后的内容上编辑；代码态改为辅助切换入口，而不是默认界面。
- 在编辑页底部增加三个操作入口：只读/编辑切换、代码模式切换、进入 Zen 模式；当用户从只读态进入代码态时，系统自动退出只读并进入可编辑源码。
- 调整主编辑页的视觉布局：编辑卡片顶部支持圆角、整体向上延伸，并压缩标题与编辑区之间的空白。
- 重做 Zen 页布局，使其只保留沉浸式写作主体和大纲侧栏；关闭 Zen 时返回进入前页面，而不是固定返回列表。
- 为普通编辑页增加悬浮式大纲展开入口，为普通编辑页与 Zen 页提供统一的大纲跳转、侧栏宽度拖拽、折叠阈值和状态持久化能力。
- 保持笔记底层存储仍为 Markdown 字符串，不改变数据库结构、自动保存链路、导出能力和标签抽取口径。

## Capabilities

### New Capabilities

- `rich-note-editing`: 主编辑页与 Zen 页共享的三态编辑模型，包括排版编辑态、只读态、代码态、模式切换规则，以及 Markdown 作为底层存储的编辑合同。
- `editor-outline-sidebar`: 主编辑页与 Zen 页的大纲与页面壳层行为，包括悬浮大纲入口、Zen 专属大纲侧栏、宽度拖拽、自动折叠和持久化恢复。

### Modified Capabilities

- 无。当前 `openspec/specs/` 尚无已生效的根规格，本次变更以新增能力的方式定义新的编辑合同。

## Impact

- 前端编辑层：`src/views/NoteEditorView.vue`、`src/views/ZenMode.vue`、`src/components/MarkdownEditor.vue` 及其测试需要重构或替换为主编辑器抽象层。
- 前端状态与设置：`src/stores/ui.ts`、`src/stores/settings.ts`、相关 composables 需要新增编辑模式、大纲栏宽度/折叠状态与 Zen 返回上下文管理。
- 编辑器能力：项目需要引入或实现“排版态可直接编辑 + Markdown 序列化”的主编辑能力，并保留源码模式切换。
- 视觉与交互：主编辑页和 Zen 页的布局、底部状态栏、悬浮按钮、大纲栏和拖拽分隔线都需要更新。
- 验证范围：需要补充模式切换、Zen 返回、侧栏拖拽/折叠/持久化和视觉布局回归测试。
