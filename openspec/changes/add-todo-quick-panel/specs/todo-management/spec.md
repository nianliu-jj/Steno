## ADDED Requirements

### Requirement: 主窗口待办路由替换
Steno 主窗口的 `todo` 路由 MUST 渲染完整的 `TodoView` 组件，取代既有的 `PlaceholderView`。该视图 MUST 包含左侧分类导航 + 右侧任务列表 + 顶部操作区三个区块。

#### Scenario: 进入待办标签页
- **WHEN** 用户从主窗口侧栏点击"待办"标签
- **THEN** 路由切换到 `/todo`，渲染 `TodoView`，URL 不再展示"即将推出"占位

#### Scenario: 路由刷新保持选中分类
- **WHEN** 用户当前选中"计划中"分类后刷新页面 / 重新进入
- **THEN** TodoView 在挂载时恢复上次选中的分类（持久化在 `useTodosStore` 或 URL query 中）

### Requirement: 任务分类侧栏
左侧分类侧栏 MUST 至少提供：今天、计划中、进行中、已暂停、已完成、全部、收件箱 七个分类项；每项 SHALL 显示该分类下未删除（`is_deleted=0`）的任务数（已完成分类计已完成数）。

#### Scenario: 分类点击切换
- **WHEN** 用户点击"进行中"分类项
- **THEN** 右侧任务列表过滤为 `status='doing' AND is_deleted=0`，分类项高亮

#### Scenario: 分类计数实时更新
- **WHEN** 用户新建 / 完成 / 删除任意任务（无论从浮窗还是主窗口）
- **THEN** 各分类计数在 ≤1 秒内重新计算并刷新显示

### Requirement: 任务列表的 CRUD
任务列表 MUST 支持完整的增删改查：行内编辑任务文本、切换状态（todo↔doing↔paused↔done）、设置 / 清除截止日期、删除（逻辑删除）。

#### Scenario: 行内编辑文本
- **WHEN** 用户双击某任务的文本区域
- **THEN** 该任务文本变为可编辑 `<input>`，失焦或按 `Enter` 后保存到数据库；按 `Esc` 取消编辑回退原文本

#### Scenario: 状态切换下拉
- **WHEN** 用户点击任务行内的状态徽章（如 "todo"）
- **THEN** 弹出下拉菜单展示四种状态，选择后立即更新数据库并刷新分类计数

#### Scenario: 设置截止日期
- **WHEN** 用户点击任务行内的日期按钮（默认显示"无日期"或现存 `due_date`）
- **THEN** 弹出日期选择器，选择日期后保存到 `due_date`，任务自动归入对应分类（"今天"或"计划中"）

#### Scenario: 删除任务
- **WHEN** 用户点击任务行末尾的删除按钮
- **THEN** 系统将 `is_deleted=1`，任务从列表移除（无二次确认）；分类计数同步减少

### Requirement: 顶部操作区
任务列表上方 MUST 提供：①与浮窗一致的"添加新任务"输入框 ②搜索框（按 `content` 模糊匹配当前分类内任务）③视图密度切换（紧凑 / 舒适）。

#### Scenario: 在主窗口添加任务
- **WHEN** 用户在顶部输入框输入"看完文档"并按 `Enter`
- **THEN** 系统创建任务（同浮窗逻辑），列表顶部出现该任务，浮窗（若已打开）也立即看到此任务

#### Scenario: 搜索过滤
- **WHEN** 用户在搜索框输入"周报"
- **THEN** 当前分类下任务列表只显示 `content LIKE '%周报%'` 的项；清空搜索框后恢复完整列表

### Requirement: 跨窗口同步
任何窗口（浮窗或主窗口）对任务的写操作 MUST 在 ≤1 秒内反映到其它已打开窗口上，且 SHALL 不需要用户手动刷新。

#### Scenario: 浮窗新增同步到主窗口
- **WHEN** 用户在浮窗添加任务"买牛奶"
- **THEN** 主窗口若处于"今天"分类，列表立即出现"买牛奶"；其它分类不显示该项（因 `due_date=NULL` 默认归"今天"）

#### Scenario: 主窗口完成同步到浮窗
- **WHEN** 用户在主窗口"今天"分类下勾选完成一个任务
- **THEN** 浮窗（若已打开）的对应任务立即从未完成区移除，"今天 N" 计数 -1
