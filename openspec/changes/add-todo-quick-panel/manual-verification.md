# 待办浮窗 add-todo-quick-panel · Phase 7 手动冒烟清单

> 启动方式：`pnpm tauri dev`。每条勾选后在 tasks.md 标记完成。

## 7.2 快捷键唤起 + focus + ESC

平台：Windows / macOS（若有）

- [ ] 应用启动后，按 `Ctrl+Shift+T`（macOS: `Cmd+Shift+T`），浮窗 ≤ 200ms 内显示
- [ ] 浮窗显示后，输入框已自动获得焦点（无需点击即可输入）
- [ ] 浮窗显示状态下再次按快捷键，浮窗隐藏
- [ ] 浮窗显示状态下按 `Esc` 键（如已绑定）应隐藏；当前未绑定 Esc，则点 × 应隐藏

## 7.3 跨窗口同步

- [ ] 主窗口左侧导航点"待办"进入 TodoView，确认渲染分类侧栏 + 任务列表
- [ ] 按快捷键唤起浮窗，在浮窗输入框输入"测试任务 A" + Enter
- [ ] 切回主窗口（不需要重新进入待办视图），"今天"与"全部"分类计数 +1，列表中可见"测试任务 A"
- [ ] 在主窗口 TodoView 中勾选"测试任务 A"标记为完成
- [ ] 切回浮窗（按快捷键唤起），未完成列表已不再显示该任务、count -1（若 includeCompleted=true 仍可见但呈划线态）
- [ ] 在主窗口删除该任务，浮窗对应项立即消失

## 7.4 快捷键冲突 + 重新绑定

- [ ] 先打开 Chrome / Edge 浏览器；浏览器自带 `Ctrl+Shift+T`（重新打开关闭标签）会占用此快捷键
- [ ] 启动 Steno 后按 `Ctrl+Shift+T`，确认能正常呼出（Steno 全局快捷键应优先；如失败，设置面板 → 待办浮窗 → 改键测试）
- [ ] 设置面板 → 待办浮窗 → 修改快捷键为 `Ctrl+Alt+Y` → 失焦或回车
- [ ] 按 `Ctrl+Alt+Y`，浮窗正常唤起
- [ ] 按旧的 `Ctrl+Shift+T`，无任何反应（旧绑定已注销）

## 7.5 启用开关 + 无重复注册

- [ ] 设置面板 → 待办浮窗 → 关闭"启用待办浮窗"开关；下方快捷键 + 位置控件应整体置灰
- [ ] 按当前快捷键，浮窗不呼出
- [ ] 重新打开"启用待办浮窗"开关；按快捷键浮窗能正常呼出
- [ ] （可选）反复开关 3 次后按快捷键，浮窗只呼出 1 次（不重复注册）
- [ ] 切到设置面板 → 待办浮窗 → 位置切换为"记住上次位置"
- [ ] 唤起浮窗，拖到屏幕左上角，按 × 关闭
- [ ] 再次唤起，浮窗在原左上角位置（lastPos 已持久化）
