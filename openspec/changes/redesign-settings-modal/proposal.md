## Why

当前设置界面仍是独立整页视图，点击主窗口“设置”后会离开当前工作台上下文，和 `docs/steno-settings-modal.html` 中定义的弹窗式设置体验不一致。现在需要把设置改为基于 Naive UI `NModal` 的模态框，让用户在不丢失主窗口工作流的情况下调整主题、快捷键、浮窗、编辑器、备份和存储偏好。

## What Changes

- 将主窗口设置入口从独立页面切换改为打开设置模态框，背景保留主窗口内容并通过遮罩隔离焦点。
- 重新设计 `SettingsView` 的信息架构，采用“顶部分类标签 + 滚动内容区 + 底部操作栏”的桌面设置面板结构。
- 设置分类覆盖常规、外观、快捷键、隐私安全、存储、关于，其中现有后端已支持的设置继续保持自动保存。
- 对尚未有后端字段的隐私安全、强调色、便签纸、应用排除名单等内容提供清晰的展示或禁用态，不在本次变更中新增持久化 schema。
- 保留并优化现有主题、快捷键、速记浮窗尺寸、失焦关闭延迟、编辑器模式、备份阈值、数据目录查看和复制能力。
- 补充设置模态框的可访问性和测试覆盖，包括弹窗可见性、分类切换、路径渲染和无 `highlight.js` 噪声。

## Capabilities

### New Capabilities
- `settings-modal-experience`: 设置模态框体验，包括打开/关闭方式、分类信息架构、现有设置项交互、存储路径展示和关于信息。

### Modified Capabilities
- 无。

## Impact

- 前端主窗口入口：`src/views/MainView.vue` 需要把设置快捷入口改为本地模态框状态，而不是通过 Tauri 打开设置窗口。
- 设置视图：`src/views/SettingsView.vue` 需要改造成可嵌入 `NModal` 的设置面板内容，并匹配原型的信息架构。
- 测试：`src/views/SettingsView.test.ts` 和主窗口相关测试需要覆盖弹窗渲染、分类切换和存储路径展示。
- 路由兼容：`settings` 窗口模式可以继续作为兜底视图存在，但主窗口点击设置时优先使用模态框。
- 依赖：不新增运行时依赖，继续使用 Vue 3、Pinia、Naive UI 和现有 Tauri IPC 封装。
