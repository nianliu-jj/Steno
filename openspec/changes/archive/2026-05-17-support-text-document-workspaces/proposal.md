## Why

Steno 目前把所有笔记都当作同一种 SQLite 记录来处理，无法表达“速记文本”和“磁盘文档”两类对象的不同生命周期，也没有工作区、文件夹和分组这些承载长期整理流程的结构。随着主窗口编辑和列表工作台已经成为用户的主入口，文本/文档分流、工作区浏览和全局分组模型需要被正式定义成下一阶段的产品合同。

## What Changes

- 把现有统一的“笔记”概念拆分为两类内容对象：`text` 和 `document`。
- `text` 由速记生成，正文保存在数据库中，并按正文 Markdown 的真实字节数限制在 `10KB` 内；超限时保存失败并返回精确体积提示。
- `document` 由主窗口“新建笔记”生成，正文保存在磁盘文件中，不受 `10KB` 限制；数据库仅保留文档索引与元信息。
- 主窗口支持把 `text` 原地转换为 `document`；转换后文档落到当前主页面所在的工作区根目录或子文件夹中。
- 引入新的结构对象：`workspace`、`folder`、`group`。工作区和文件夹承载文档层级；分组承载文本层级，且文本只能属于一个分组。
- 主窗口列表页改成“混合内容视图”：当前工作区/文件夹下的文件夹与文档，和全局分组与文本同时显示；右侧结构栏只显示当前工作区的文件夹/文档树。
- 主窗口底部栏新增当前工作区显示、当前页面统计和右侧结构栏展开入口。
- 笔记卡片右上角增加类型标识：文件夹、分组、文档、文本。
- 右键菜单新增“转为文档”，只对 `text` 生效；`document` 上该动作不生效。
- 筛选下拉列表新增持久化类型筛选项：文件夹、分组、文档、文本。

## Capabilities

### New Capabilities

- `text-document-entry-model`: 文本与文档的条目类型、存储边界、大小限制、默认分组和文本转文档流程。
- `workspace-group-browser`: 工作区、文件夹、分组的层级关系，混合内容视图，当前工作区上下文和右侧结构栏行为。
- `main-list-type-filtering`: 主列表卡片类型标识、右键动作可用性、底部统计信息和持久化类型筛选。

### Modified Capabilities

- 无。当前 `openspec/specs/` 仍没有已生效的根规格，本次继续以新增能力的方式建立新的内容模型与工作区合同。

## Impact

- 数据模型：`src-tauri/src/models.rs`、`src/types/steno.ts`、SQLite schema 与存储接口需要从“单一 notes 表”扩展为“统一条目索引 + 文档磁盘正文”。
- 主窗口交互：`src/views/MainView.vue`、主列表筛选、卡片右键菜单、底部栏和右侧结构栏都需要围绕工作区/分组混合模型重组。
- 速记与新建流程：`quicknote` 默认落文本，主窗口“新建笔记”默认落文档，现有保存链路与错误提示需要分流。
- 文件系统能力：需要引入工作区目录、文件夹导航、文档文件读写和文本转文档落盘。
- 测试范围：需要新增文本 `10KB` 限制、文档落点、工作区/分组上下文、类型筛选、卡片标识和转换动作的回归测试。
