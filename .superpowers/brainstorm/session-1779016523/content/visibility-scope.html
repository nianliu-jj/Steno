先确认一个最关键的模型边界

我目前理解：文档属于工作区/文件夹树；文本和分组是全局的，不属于任何工作区。现在需要确认“当用户正停留在某个工作区或文件夹时，中间列表应该显示什么”。

当前理解的对象关系
磁盘侧

工作区

工作区 A
├─ 文件夹 1
│  ├─ 文档 a.md
│  └─ 文档 b.md
└─ 文件夹 2
   └─ 文档 c.md
数据库侧

全局分组 / 文本

分组 A
├─ 子分组 A-1
└─ 文本 1 / 文本 2

分组 B
└─ 文本 3
A

推荐：工作区视图只看当前工作区树

当选中工作区或文件夹时，中间列表只显示该工作区/文件夹下的文件夹与文档；文本和分组作为全局对象，只在“分组视图 / 文本视图 / 全部内容”里出现。

B

混合：工作区视图里继续混显示全局文本

当选中工作区或文件夹时，中间列表既显示该工作区下的文件夹与文档，也继续混入全局文本 / 分组，只是它们本身不受工作区切换影响。

我为什么更推荐 A

  • 工作区和分组的边界最清楚，不会把磁盘对象和数据库对象混在一个容器里。
  • 右侧树、统计信息、筛选项和“切换工作区”按钮都更容易解释。
  • 后面做“转为文档”时，文档落到哪个工作区/文件夹也更自然。

如果选 B 的代价

  • 同一列表里会同时出现磁盘层级对象和全局对象，排序、统计和筛选会更复杂。
  • 用户更难理解“为什么这个文本不属于当前工作区，却还在这里”。
